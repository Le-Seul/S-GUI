配置文件GUI_Config.h

GUI_Config.h文件包含S-GUI的主要配置信息，这些配置通过若干个宏来控制源代码中的条件编译选项，因此，GUI_Config.h中的配置是静态的，它们在编译时就已经确定，无法在程序执行时修改。GUI_Config.h文件在 /GUI/Config 目录下。用户可以通过修改这个宏来实现对S-GUI的裁剪，不过目前的版本的S-GUI裁剪余地很小。

功能配置宏

GUI_MSG_QUEUE_SIZE

这个宏定义了S-GUI的消息队列允许的容量。S-GUI使用固定大小的消息队列。将GUI_MSG_QUEUE_SIZE设置的更大可容纳更多的的消息，但是也会占用更多的RAM。

GUI_DEF_FONT

这个宏定义了S-GUI的默认字体，其值的类型为GUI_FONT。

GUI_MAX_MSG_NEST

最大消息嵌套层数，函数WM__SendMessage()在时会对消息嵌套层数计数，如果嵌套超过GUI_MAX_MSG_NEST层, S-GUI将会停止递归并产生一个致命错误。如果栈空间比较大并且需要更多的消息嵌套层数则可以把GUI_MAX_MSG_NEST设置的大一些。

GUI_USE_MEM_LOCK

是否使用内存管理单元的互斥锁，如果用户需要在多线程环境中单独调用GUI_Malloc()和GUI_Free()函数就需要定义GUI_USE_MEM_LOCK为1，否则使其为0.这样可以少用一个互斥锁，并且提高运行速度。

GUI_DEBUG_MODE

调试模式开关GUI_DEBUG_MODE非0会打开调试模式。开启调试模式时，如果S-GUI在运行中发生错误，将会通过用户实现的调试输出接口输出相关的调试信息。当然，有很多错误都无法通过调试模式捕获，这时还需要结合开发工具的调试功能来进行调试。
