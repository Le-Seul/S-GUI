开始

获取源代码

您可以通过访问https://github.com/Le-Seul/S-GUI页面来下载S-GUI的源码。如果系统安装了git工具，也可以通过命令: git clone https://github.com/Le-Seul/S-GUI.git 将源代码克隆到本地。下载的S-GUI目录下包含完整的S-GUI库以及一个demo程序，源代码在Codes目录下面，demo的工程文件或者编译脚本在build目录下。

编译

Windows:

Visual Studio 2015:

使用Visual Studio 2015打开文件文件build/msvs/S-GUI.sln，点击"本地Windows调试器"按钮即可编译并开始运行demo程序。

GCC:

若要在Windows下使用GCC编译S-GUI的demo程序，需要确保已经安装了GCC并且设置好环境变量，此时应该可以在cmd窗口下使用gcc和make命令，如果您安装的GCC版本没有gcc.exe和make.exe文件，请到GCC安装目录下把实现对应功能的可执行文件重命名成gcc.exe或者make.exe。一切准备就绪之后，按下Win + R键后输入"cmd"进入cmd窗口，然后使用cd指令打开build/mingw目录，最后使用make命令来编译。具体命令如下(以E:/S-GUI目录为例): 01 E:
02 cd E:\S-GUI\build\mingw
03 make 编译成功后会在build/mingw目录下生成demo.exe文件，到Codes/sdl/win32目录下将SDL.dll文件复制到build/mingw目录下之后就可以运行demo.exe了。此外，可以通过make clean命令删除编译产生的中间文件，不过这需要确保你的系统能够使用rm命令。

Linux:

在Linux下编译demo程序需要预先安装SDL库，首先到http://www.libsdl.org/release/SDL-1.2.15.tar.gz下载SDL 1.2的源代码，解压后执行以下命令: 01 cd ./SDL-1.2.15
02 ./configure
03 make
03 sudo make install 然后进入S-GUI源代码的build/linux目录，执行: 01 make
02 ./demo 就可以编译个运行S-GUI的demo程序了。可以用make clean命令删除编译产生的中间文件。
